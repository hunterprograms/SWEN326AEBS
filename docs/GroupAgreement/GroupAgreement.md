## 1. Communication

- Primary communication channel: **Discord**
- All group members are expected to respond to Discord messages within **24 hours**
- Important decisions must be confirmed in Discord in writing so there is a record
- Questions for the course coordinator must be posted on the **Nuku forum**, not emailed directly

---

## 2. Meetings

- The group will meet **thrice per week** at a time agreed by all members
- Twp of these meetings will be held online via Discord voice/video
- A brief summary of each meeting (decisions made, tasks assigned) will be posted in Discord and committed to the repo under `docs/GroupAgreement/MeetingNotes/`
- If a member cannot attend, they must notify the group **at least 2 hours in advance** and catch up async via the meeting notes

- The third and final meeting will be held in person
- This meeting will be dedicated to group development 

---

## 3. Task Allocation

- Tasks will be tracked via **GitLab issues**
- At each meeting, tasks will be assigned and logged as GitLab issues
- Each issue should reference the relevant requirement ID (e.g. REQ-001) where applicable
- Members should self-assign issues they pick up and close them with a commit reference when done

## 5. Code Standards

- Language: **Java**
- All code must follow the **Power of Ten** coding rules as specified in SWEN326
- Every class/method should include a comment referencing its relevant requirement (e.g. `// REQ-006`)
- Static analysis tools should be run before committing code
- Commits should be small, descriptive, and frequent

---

## 6. Handling Delays and Broken Agreements

- If a member cannot complete a task on time, they must notify the group **as soon as they know**, not at the deadline
- The group will collectively decide how to redistribute the work
- Repeated failure to contribute or communicate will be noted in individual reports under "Comments on Group Members"
- This is not a blame document — it is an acknowledgement that life happens, and early communication is the only ask
